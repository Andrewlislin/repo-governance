# repo-governance v1.1 implementation plan

Version 1.1 keeps the repo-governance CLI as the only policy engine and adds explicit automation for existing, new, and cloned repositories. Presets are static policy inputs; Agents consume CLI JSON and never reimplement RG001–RG005.

Implementation is split into five Coding changes after this initializer:

1. Preset-driven `bootstrap`, adoption checks, hook composition, and stable reports.
2. Safe `new` and `clone` entry points with offline command-level E2E tests.
3. Deterministic `prepare-pr` projection and first-push diagnostics.
4. Agent-neutral playbooks and thin Codex Skills.
5. Claude Code adapters and cross-agent contract fixtures.

The optional GitHub-formatting enhancement is intentionally outside the v1.1 feature ledger until the first five changes are stable.

## Fixed boundaries

- No transparent interception of native `git clone`.
- No policy selection without an explicit preset.
- No silent GitHub write or ruleset mutation.
- No Agent-side rule engine.
- Local hooks, CI, and Agent reports use the same version-pinned CLI.
