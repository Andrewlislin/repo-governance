# v1.3 release

Version 1.3 adds RG006 and execution-contract version 1 without changing the repository configuration schema version. Release and installed engine manifests publish pre-push protocol version 1 and the supported execution-contract versions.

The stable dispatcher selects the exact candidate commit’s installed engine and blocks missing, downgraded, incompatible, or digest-mismatched identities. Pre-push validates each unique pushed tip/base pair in an isolated local clone. CI binds dynamic execution to the event head/base revision and produces clean-checkout evidence only after static validation, runtime verification, governed dependency preparation, profile execution, and final revision checks.

Repository Hook connection now uses a manifest-verified wrapper and optional original-Hook sidecar. Existing Hooks receive the complete captured stdin before the dispatcher; `hooks disconnect` restores exact original bytes only when no component changed.

Migration is engine-first: install a verified v1.3 engine and stable launcher, run `repo-governance hooks connect`, then commit the exact engine identity, execution contract, protected workflow, tests, and documentation together. There is no compatibility warning period or fallback to the old `check` path.

The protected `pr-validation` profile prepares dependencies with `npm ci --ignore-scripts`, then invokes the public `npm run ci:pr` entry. The repository-required check is `Repo Governance / validate`; it is produced only by the exact-SHA Action in `.github/workflows/repo-governance.yml`.
